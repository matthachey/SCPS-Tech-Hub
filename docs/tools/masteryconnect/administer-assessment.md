# Administering Assessments to Students

Once your assessment is created, you need to get it in front of students. MasteryConnect offers several ways to administer assessments — this guide covers the most common methods used at SCPS.

**Time to complete:** ~5 minutes

---

## Method 1: Testing code (most common)

A testing code is a unique code students enter to access the assessment on their devices.

### Generate a testing code

1. Open your Mastery Tracker and go to the **Assessments** tab.
2. Find the assessment you want to administer.
3. Click on the assessment, then click **"Administer"** or **"Generate Code"**.
4. A **testing code** will appear (a short alphanumeric code like `A3X7K2`).

### Share the code with students

1. Display the code on your projector, whiteboard, or share it verbally.
2. Students go to **[masteryconnect.com](https://masteryconnect.com)** and click **"Enter Testing Code"** (or they may have a student portal bookmark).
3. They enter the code and begin the assessment.

!!! tip "Managing the testing window"
    - You can **generate a new code** after class to prevent students from continuing the assessment outside of class time.
    - If a student gets disconnected, their progress is saved — they can re-enter the code to resume.

---

## Method 2: Through Canvas

If your tracker is linked to Canvas, you can push assessments directly into your Canvas course.

1. When creating or administering the assessment, look for the option to **"Push to Canvas"** or **"Create Canvas Assignment"**.
2. The assessment will appear as an assignment in your Canvas course.
3. Students access it just like any other Canvas assignment.

!!! info "Why use Canvas delivery?"
    - Students don't need a separate testing code
    - The assessment appears in their familiar Canvas workflow
    - Grades can sync back to Canvas (depending on your settings)

---

## Method 3: Paper-based with bubble sheets

For situations where devices aren't available or you prefer paper administration:

1. In the assessment builder, make sure **"Printable"** is enabled.
2. **Print the assessment** for students.
3. **Download bubble sheets** from MasteryConnect — these come pre-slugged with student ID numbers.
4. Students complete the bubble sheets by hand.
5. See [Scoring with bubble sheets](bubble-sheets.md) for scanning and scoring instructions.

---

## During the assessment

### What you can see

While students are testing, you can monitor progress from your tracker:

- Which students have **started** the assessment
- Which students have **submitted** the assessment
- Real-time **completion counts**

### Managing student issues

| Issue | Solution |
|-------|----------|
| Student can't enter the code | Verify the code is correct and the assessment is active |
| Student got kicked out mid-test | Their progress is saved — have them re-enter the code |
| Student needs more time | The assessment stays open as long as the code is active |
| Student submitted by accident | Contact your ITRT — submissions can sometimes be reopened |
| Wrong student took the assessment | Note it and contact your ITRT for data correction |

---

## After the assessment

Once all students have submitted:

1. **Auto-scored items** (multiple choice, true/false) will have results available immediately.
2. **Manually scored items** (short answer, essay) will need to be scored in the **Performance Grader**.
3. Navigate to your Mastery Tracker to see the updated data.

!!! info "Performance Grader tip"
    When scoring in the Performance Grader, make sure to click **"Save"** for each student before clicking **"Next"**. There is a known issue where the next arrow may not appear until you save first.
