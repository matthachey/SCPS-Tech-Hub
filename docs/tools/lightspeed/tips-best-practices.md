# Lightspeed Tips and Best Practices

A collection of power-user strategies, common mistakes to avoid, and SCPS-specific tips for getting the most out of Lightspeed Classroom.

**Time to complete:** ~5 minutes

---

## The Lightspeed mindset

!!! tip "Key principle"
    Lightspeed is a **teaching tool**, not a surveillance tool. The goal is to support learning, not to catch students doing something wrong. Use it to redirect, differentiate, and engage — not to police.

---

## Top strategies from experienced teachers

### 1. Tell students you use it

The single most effective strategy is **transparency**. On the first day, show students what you see. Demonstrate the screen monitoring, the message feature, and the tab history. When students know the tool exists, off-task behavior drops dramatically — and you barely have to use it.

### 2. Message before you lock

When you see a student off-task, try this escalation:

1. **Send a message** — "I see you're not on the assignment. Need help?"
2. **Close the tab** — remove the distraction
3. **Lock their screen** — only if the behavior continues
4. **Talk to the student in person** — private conversation

Most students self-correct at step 1.

### 3. Build your Allow Lists in advance

Don't wait until class starts to set up your web rules. Before the week begins, build out your Allow Lists for each major activity. Then it's just a toggle during class.

### 4. Use groups for stations or rotations

If you run station-based learning, create a group for each station. Push the correct link and set the correct web rules for each group. Students rotate through stations and their access changes with them.

### 5. Check history, not just live screens

A quick scan of browsing history at the end of class tells you more than watching screens all period. You can see patterns of off-task behavior over time and address them in a targeted conversation.

## Common mistakes to avoid

!!! warning "Things to watch out for"
    - **Leaving a session running after class ends.** This blocks the next teacher from monitoring those students. Always end your session.
    - **Using Allow Mode without enabling SSO.** Students won't be able to log into Google tools, and you'll get a flood of "it's not working" complaints.
    - **Setting web rules for the whole class when only one student needs them.** Use individual actions or groups instead.
    - **Over-locking screens.** Students lose engagement and momentum when locked too long or too often. Use it sparingly.
    - **Forgetting to include `docs.google.com` in Allow Lists.** Almost every assignment involves Google Docs — if it's not on the list, students can't work.

## SCPS-specific reminders

| Topic | Details |
|-------|---------|
| **Roster sync** | Lightspeed syncs from Google Classroom. If a student isn't showing up, check their Google Classroom enrollment first. |
| **Off-campus monitoring** | Teachers cannot monitor students outside of school hours or off campus. The district filter still works everywhere, but Lightspeed Classroom is school-only. |
| **District filter vs. your rules** | Your Allow/Block lists work on top of the district filter. You can restrict access further than the district filter, but you can't unblock something the district has blocked. |
| **Requesting district-level changes** | If a site needs to be allowed or blocked at the district level, submit a ticket through your building's IT process. |
| **Lightspeed AccessScan** | This Chrome extension (added to teacher accounts) lets you check if a specific URL is allowed or blocked by the district filter before class. Use it to avoid surprises. See the [full AccessScan guide](accessscan.md). |

## Quick reference: Feature cheat sheet

| I Want To... | Do This |
|--------------|---------|
| See all student screens | Open your class — they're right there |
| Zoom in on one student | Click their thumbnail |
| Send a message | Top toolbar → Message |
| Push a link | Top toolbar → Push Link |
| Share my screen | Top toolbar → Share Screen |
| Lock all screens | Top toolbar → Lock Screens |
| Block a website | Web Rules → Block Mode → add URL |
| Allow only specific sites | Web Rules → Allow Mode → add URLs |
| Create a student group | Hover under My Class → Create Group |
| Check browsing history | Click student → History |
| Close an off-task tab | Click student → find tab → Close |
